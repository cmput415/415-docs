Change log

  • 2020/11/18 19:00

    • Clarified when memory should be freed. (Memory Management)

    • Clarified that typedef type symbol names do not conflict with variable or subroutine symbol names. (Typedef, Section 12)

    • Clarified that null and identity cannot be cast. (Null and Identity, Section 10.7)

    • Clarified that “two sided” promotion may occur with tuples. (Tuple to Tuple, Section 11.4)

    • Clarified that return statements must have values compatible with the type in the returns clause. (Return, Section 14.7)

    • Clarified that procedures without returns clauses do not require explicit return statements. (Return, Section 14.7)

    • Clarified that scalar character values can still take part in the concatenation (||) operation with a string or vector with type character. (Operations, Section 7.2.5)

    • Rewrote several sections of string:

      • Clarified differences in type description. (String, Section 7.8)

      • Clarified declaration methods. (Declaration, Section 7.8.1)

      • Clarified concatentation of characters onto strings. (Operations, Section 7.8.5)

    • Clarified that whitespace cannot be in a real literal when in the input stream. (Input Format, Section 4.2.1)

    • Fixed example of misreading a boolean from the std_input. (Input Format, Section 4.2.1).

    • Added restrictions on input stream rewinding. (Stream State, Section 18.4; Error Handling, Section 4.2.2)

    • Moved and expanded description of stream rewinding. (Stream State, Section 18.4; Error Handling, Section 4.2.2)

    • Removed “and subsequent reads” due to ever-expanding nature of std_input. (Error Handling, Section 4.2.2)

    • Trimmed information in description of stream_state which was duplicating the description of streams. (Stream State, Section 18.4)

  • 2020/10/22 16:20

    • Cleaned up latex artifacts messing up a code block. (Return, Section 14.7)

    • Changed pythag function so that the exponent is a real. (Syntax, Section 15.1)

    • Fixed latex artifact where character literal for ' was \’ not \'. (Literals and Escape Sequences, Section 7.2.4)

    • Removed usages of std_output as an assignable value. (Main, Section 16.3; Tuple to Tuple, Section 11.4; Operations, Section 7.8.5; Scalar to Vector or Matrix, Section 11.2)

    • Remove mention of matrix keyword that no longer exists. (Declaration, Section 7.9.1)

    • Clarify lack of = <stmt>; format for procedures. (Syntax, Section 16.1)

    • Fix usage of if ... fi that is not used in Gazprea. (Operations, Section 7.5.6).

    • Remove usage of 0.0f which is C syntax. (Assignment Statements, Section 14.1)

    • Fix malformed types for vectors and matrices where the size was not attached to the type. (Operations, Section 7.7.5; Declaration, Section 7.9.1)

    • Clarified format of real literals without scientific notation. (Literals, Section 7.4.4)

    • Clarified format of real literals with scientific notation. (Literals, Section 7.4.4)

  • 2020/09/01 15:00

    • Initial release for Fall 2020
