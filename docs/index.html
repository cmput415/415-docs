New in Fall 2020: Fuzzers for Generator, SCalc and VCalc

New in Fall 2020 is the addition of fuzzers for the Generator, SCalc and VCalc assignments.

Fuzzers are tools used in a method of software testing called fuzz testing, where a program is fed random inputs in hope of finding software bugs.

You can essentially view fuzzers as random test-case generators for helping you find bugs in the compilers and interpreters that you create.

Fuzzers for the Generator, SCalc and VCalc can be found on the CMPUT 415 GitHub or by visiting the Testing sections of the assignment specifications.

Fuzz testing is an active area of research that has broad applications in finding compilers bugs, security vulnerabilities, etc. There are many approaches to fuzz testing; all varying in sophistication.
In the following readings you will explore two different fuzzers, CSmith and NAUTILUS, which each have their own take on how to approach fuzzing.

Readings:

  1. Yang, X., Chen, Y., Eide, E., & Regehr, J. (2011). Finding and Understanding Bugs in C Compilers. ACM SIGPLAN Conference on Programming Language Design and Implementation (PLDI), June 2011, San Jose, CA, USA.
  2. Aschermann, C., Frassetto, T., Holz, T., Jauernig, P., Sadeghi, A., & Teuchert, D. (2019). NAUTILUS: Fishing for Deep Bugs with Grammars. Network and Distributed Systems Security (NDSS) Symposium, February 2019, San Diego, CA, USA.
